Responsivitas & Mobile-Friendly 3 isu
Viewport meta tag sudah ada
Tag width=device-width, initial-scale=1 terpasang dengan benar di semua halaman.
Logo marquee duplikat berlebihan
Logo instansi (DKI, PUPR, Dishub, DLH) diulang 28+ kali di DOM. Di mobile ini memperlambat render dan membuang bandwidth.
Saran: Gunakan CSS animation infinite pada ~8 item saja, bukan clone ratusan elemen
Halaman /login-masyarakat kosong
Halaman login tidak merender konten apapun — kemungkinan ada bug pada conditional rendering atau route yang belum selesai.
Saran: Tambahkan fallback UI / loading state agar halaman tidak tampak blank
Anchor link navbar menuju section di beranda
Link seperti #info-section di navbar pada halaman /progress dan /kontak-darurat tidak akan scroll ke mana pun karena section itu tidak ada di halaman tersebut.
Saran: Arahkan link navbar ke /#info-section (dengan prefix URL beranda)
Performa & Load 2 isu
Deploy di Vercel
Vercel otomatis memberikan CDN global, HTTPS, dan kompresi — fondasi performa sudah solid.
Gambar menggunakan path lokal
Aset gambar di-host sendiri (bukan CDN eksternal pihak ketiga) sehingga tidak ada ketergantungan eksternal yang berisiko lambat.
Logo marquee membebani DOM
28+ elemen <img> identik di-load untuk efek scroll logo. Ini memperberat DOM dan memicu banyak request gambar berulang.
Saran: Gunakan max 1 set logo (4 item) + CSS infinite scroll animation
Gambar tidak ada atribut loading="lazy"
Gambar di halaman /progress (perbaikan jalan, PJU, jembatan) tidak ada lazy loading, sehingga semua diunduh saat halaman pertama kali dibuka.
Saran: Tambahkan loading="lazy" pada semua img yang berada below the fold
Aksesibilitas (a11y) 5 isu
Semua gambar logo marquee tanpa alt yang berbeda
Ratusan tag <img> logo duplikat memiliki alt text yang sama persis — screen reader akan membacanya ratusan kali berturut-turut.
Saran: Kurangi elemen DOM dan tambahkan aria-hidden="true" pada elemen duplikat dekoratif
Icon Material Symbols tidak ada aria-label
Icon teks seperti visibility, map, verified, ambulance dirender sebagai teks literal — screen reader akan membacanya sebagai kata "ambulance", "map", dll. secara aneh.
Saran: Tambahkan aria-hidden="true" pada semua icon dekoratif, atau gunakan tag <span aria-hidden>
Tombol "Masuk Portal" dan "menu" tidak ada konteks
Teks "menu" di navbar kemungkinan adalah tombol hamburger tanpa aria-label yang menjelaskan fungsinya.
Saran: Tambahkan aria-label="Buka menu navigasi" pada tombol hamburger
Statistik angka tidak ada konteks unit
Angka seperti "92%", "340+", "15mnt", "15,310" muncul tanpa label yang semantik — tidak jelas untuk screen reader apa satuan/konteksnya.
Saran: Gunakan <dl><dt><dd>> atau tambahkan aria-label pada container statistik
Link "Tracking Laporan", "Lihat Peta Kota" semua menuju URL yang sama
Tiga CTA berbeda semuanya menuju /login-masyarakat — padahal teksnya menjanjikan fitur berbeda. Ini membingungkan pengguna dan screen reader.
Saran: Pastikan setiap link menuju halaman/fitur yang relevan, atau jelaskan bahwa login diperlukan lebih dulu
SEO & Meta Tags 3 isu
Title & meta description sudah ada
Semua halaman memiliki <title> dan meta-description yang konsisten dan deskriptif.
Struktur heading sudah baik
Setiap halaman menggunakan H1 yang jelas dan H2/H3 yang hierarkis dengan konten yang relevan.
Title & description sama di semua halaman
Halaman /progress dan /kontak-darurat memiliki title dan meta description yang identik dengan beranda — tidak optimal untuk SEO.
Saran: Buat title unik per halaman, misal "Progres Perbaikan Infrastruktur | SIGAP"
Tidak ada Open Graph tags
Ketika link SIGAP dibagikan di WhatsApp, Twitter, atau LinkedIn, tidak akan ada preview gambar/deskripsi yang menarik.
Saran: Tambahkan og:title, og:description, og:image, og:url di setiap halaman
Tidak ada sitemap.xml / robots.txt
Tidak terdeteksi sitemap — ini memperlambat indexing oleh Google, terutama untuk halaman /progress dan /kontak-darurat.
Saran: Generate sitemap.xml dan tambahkan robots.txt (Next.js bisa generate otomatis)
Kode & Best Practices 3 isu
Menggunakan Next.js
Framework modern dengan SSR/SSG, routing, dan optimasi bawaan — pilihan yang tepat untuk project ini.
Deploy otomatis via Vercel
CI/CD sudah terhubung — perubahan kode otomatis ter-deploy tanpa konfigurasi tambahan.
Halaman /login-masyarakat tidak merender apapun
Kemungkinan ada error pada komponen, conditional rendering yang salah, atau halaman belum selesai dikerjakan. Ini adalah bug kritis karena banyak CTA mengarah ke sini.
Saran: Debug halaman ini — cek console error dan pastikan ada fallback UI
Anchor link tidak cross-page aware
Navbar menggunakan hash links (#section) tanpa prefix URL beranda, sehingga broken saat diakses dari halaman lain.
Saran: Ganti jadi href="/#info-section" agar bekerja dari halaman manapun
Konten footer menggunakan icon font sebagai teks
Icon seperti location_on, phone, mail dirender sebagai plaintext — ini berarti menggunakan Material Icons font, tapi perlu dipastikan sudah di-load dengan benar di semua browser.
Saran: Pastikan Material Icons dimuat, atau ganti dengan SVG icon yang lebih reliable
